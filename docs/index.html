[ 01_THE_PAIN_POINT ]

Three tools. Three syntaxes. Three failure modes. Zero consistency.

Pain Point winget chocolatey scoop
Search syntax `winget search` `choco search` `scoop search`
Install syntax `winget install` `choco install` `scoop install`
Output format Column-based, locale-dependent Pipe-delimited (`|`) Bracket-based or columnar
Silent failure mode Cryptic HRESULT codes Exit code 1 for everything Bucket not found, no error
Requires admin? Sometimes Almost always Never

[ 02_COMMAND_REFERENCE ]

`pacwin` speaks two dialects: pacman-style flags for muscle memory, and verbose commands for readability.

Task Verbose pacman-style
Searchpacwin search <query>pacwin -Ss <query>
Installpacwin install <id>pacwin -S <id>
Uninstallpacwin uninstall <id>pacwin -R <id>
Update allpacwin updatepacwin -Syu
List installedpacwin listpacwin -Q
Check outdatedpacwin outdatedpacwin -Qu
Hold / Pinpacwin hold <id>pacwin pin <id>

[ 03_ARCHITECTURE_&_SECURITY ]

No-nonsense engineering. Zero bloat. Windows-first architecture.

integration_instructions

CONCURRENCY ENGINE

`RunspacePool` (threads) on PS 5.1 to avoid `Start-Job` overhead. `ForEach-Object -Parallel` on PS 7+. Same process, shared memory.

security

SECURITY MODEL

Strict regex input sanitization. `Set-StrictMode -Version 2.0`. No `Invoke-Expression`. Ever.

inventory_2

ZERO_DEP_INIT

Stand-alone script module (`.psm1`). No compiled binaries, no DLL hell.
