💡 **What:** Replaced the array append (`+=`) operations in `pacwin.psm1` inside of `_pw_do_export` and `_pw_do_import` with instances of `[System.Collections.Generic.List[object]]` and `[System.Collections.Generic.List[string]]` respectively.

🎯 **Why:** In PowerShell, using `+=` on an array creates a brand new copy of the array behind the scenes on every iteration, leading to O(N^2) time complexity and excessive garbage collection overhead. Since the task requested optimizing loop operations involving arrays, I upgraded these specific collections to use mutable `List` classes that provide an `O(1)` amortized `.Add()` method which is significantly faster and more memory efficient when adding multiple elements.

📊 **Measured Improvement:** As noted during the attempt to verify functionality, the development environment in this context lacks an installation of `pwsh` or `powershell`. Thus, it is impractical to establish a benchmark baseline or measure the exact impact locally. However, avoiding `+=` in loops is an established PowerShell performance optimization pattern guaranteed to drastically reduce operations time when arrays scale (from O(N^2) down to O(N) linear time overall for an N-element array).
